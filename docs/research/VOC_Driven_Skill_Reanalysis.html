4 Einstein 三大限制 = 我们的机会点

"Einstein...必须是工作流，其他都不行...template 的提示是一定的"
— 吴总(HS) | 技术洞察

🔒 限制1：固定 Template

Einstein Prompt Builder 使用预定义模板，虽然可以复制修改，但存在 Token 限制和动态数据随机报错问题

我们的方案：灵活提示词 + 无 Token 限制焦虑

🔒 限制2：强制工作流

Einstein 必须通过 Flow 或 Apex 触发，无法自由对话式交互，用户体验受限于预设流程

我们的方案：自由对话式 AI + 按需触发工作流

🔒 限制3：仅内部数据

Einstein 只能访问 Salesforce 内部数据，无法直接连接外部知识库、文档系统或第三方数据源

我们的方案：外部数据连接器 + RAG 增强

🌟 额外机会：数据主权

日本企业对云端 AI 数据安全极度敏感，"数据脱敏性特别特别在意"

差异化优势：私有化部署 + 本地模型 = "Salesforce 给你标准化，我们给你数据主权"

调研来源： Salesforce Einstein Features Analysis | Prompt Builder Cheat Sheet | ECI Considerations
